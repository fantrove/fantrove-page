### New

- **Fullscreen Popup — ประเภท popup ใหม่แบบเต็มหน้าจอ (PopupSystem.fullscreen)**
  ประเภท popup ใหม่ที่ครอบคลุมทั้งหน้าจอ (100vw x 100vh) ด้วยเลย์เอาต์สะอาดตาและเข้ากับแอป  เหมาะสำหรับแผงเนื้อหาที่ซับซ้อน รายละเอียดข้อผิดพลาด หน้าค้นหา หรือเนื้อหาใดๆ ที่ต้องการพื้นที่เต็มหน้าจอ รองรับการแสดง/ซ่อนหัวข้อ (showHeader) โหมดเลย์เอาต์เนื้อหา 2 แบบ ('fit' สำหรับเลื่อนภายใน, 'stretch' สำหรับเต็มความสูง) และรองรับปุ่มย้อนกลับของ browser ผ่าน History API (hideOnBack) ใช้แอนิเมชันแบบ opacity เพียวเพื่อการเปลี่ยนผ่านที่นุ่มนวลเหมือนแอปจริง ทำงานที่ z-index 28000 ชั้นสูงสุดในระบบ popup

- **showErrorFullscreen() — ระบบแสดงข้อผิดพลาดแบบรวมศูนย์**
  ฟังก์ชันใหม่ในระบบ nav-core ที่รวบรวมการแจ้งเตือนข้อผิดพลาดที่กระจัดกระจายทั้งหมด มาแสดงเป็น fullscreen popup เดียวที่สม่ำเสมอ ทุกข้อผิดพลาดถูกแสดงในโครงสร้างที่ชัดเจน ประกอบด้วยข้อความผิดพลาด ประเภทของข้อผิดพลาด เวลาที่เกิด (ทั้งภาษาไทยและอังกฤษ) รายละเอียด error object แบบเต็ม และปุ่ม "คัดลอกรายละเอียดข้อผิดพลาด" ที่เด่นชัด รองรับป้ายกำกับภาษาไทยและอังกฤษอัตโนมัติตามภาษาปัจจุบัน มี CSS injection แบบ inline เพื่อความสมบูรณ์ในตัวเอง และ fallback เป็น toast หาก PopupSystem ยังไม่พร้อมใช้งาน

### Improved

- **Error Boundary ใช้ Fullscreen Popup แล้ว (performance.js)**
  ระบบจับข้อผิดพลาดระดับ global ที่คอยจับ JavaScript error ที่ไม่ได้รับมือ ได้รับการอัพเกรดให้ใช้ `showErrorFullscreen()` แทน `showNotification()` แบบเดิม ข้อผิดพลาดถูกนำเสนอพร้อมรายละเอียดครบถ้วนและบริบทที่ชัดเจน ทำให้การ debug ง่ายขึ้นอย่างมาก มี throttle 2 วินาทีเพื่อป้องกัน popup ข้อผิดพลาดเด้งซ้ำเมื่อเกิด error หลายตัวติดต่อกัน

- **การแสดงข้อผิดพลาดของ Navigation (router.js)**
  ความล้มเหลวในการนำทางและ routing error ที่ก่อนหน้านี้แสดงเป็น toast เล็กๆ ที่มองข้ามง่าย ตอนนี้ถูกแสดงเป็น fullscreen popup ผู้ใช้เห็นภาพชัดเจนว่าอะไรเกิดขึ้นระหว่างการนำทาง พร้อมสามารถคัดลอกรายละเอียด error เพื่อรายงานได้

- **การแสดงข้อผิดพลาดของ Data Fetching (data.js)**
  เมื่อการโหลดเนื้อหาล้มเหลวหลังใช้ retry ครบทุกครั้ง ข้อผิดพลาดจะถูกแสดงเป็น fullscreen popup แทน toast ที่ปิดได้ง่าย รับรองว่าผู้ใช้ได้รับการแจ้งเตือนอย่างเหมาะสมเกี่ยวกับความล้มเหลวในการโหลดข้อมูลและสามารถดำเนินการได้ (เช่น คัดลอก error เพื่อแจ้ง support)

- **การจัดการข้อผิดพลาดในการเริ่มต้นระบบ (init.js)**
  บล็อก catch ทั้ง 3 จุดในโมดูลเริ่มต้นระบบ nav-core ได้รับการอัพเดทให้ใช้ `showErrorFullscreen()` ข้อผิดพลาดใดๆ ที่เกิดระหว่างช่วง bootstrap ของระบบ navigation จะถูกนำเสนออย่างชัดเจน แทนที่จะแสดงเป็น toast ที่หายไปเร็ว

### Fixed

- **แก้ไข Popup แจ้งอัพเดทเวอร์ชั่นไม่แสดง**
  popup แจ้งอัพเดทเวอร์ชั่น (ที่แสดงเมื่อมีการ deploy เวอร์ชั่นใหม่) เสียมานาน สาเหตุคือ `version-core.js` เรียก `PopupSystem.container()` ซึ่งเป็นเมธอดที่ไม่มีอยู่ใน PopupSystem API เมธอดที่ถูกต้อง `PopupSystem.open()` มีอยู่ตลอดแต่ไม่เคยถูกเชื่อมต่อ ตอนนี้ได้รับการแก้ไขแล้ว popup แจ้งอัพเดทเวอร์ชั่นจะแสดงถูกต้องเมื่อตรวจพบเวอร์ชั่นใหม่ โดยใช้ popup ประเภท dialog มาตรฐานพร้อมเนื้อหาและสไตล์แจ้งอัพเดทแบบ custom
