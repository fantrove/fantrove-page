### New

- **PopupSystem v2.0 — universal popup shell**
  A complete popup infrastructure with shadow DOM isolation, 8 presets (alert, confirm, toast, etc.), event bridge, and theme support. All popup-related UI now uses this system instead of inline implementations.

- **version-core.js — update notification popups**
  Users now see a styled popup notification when a new version is available. The popup shows version number, update summary, up to 4 change items, and a link to the What's New page. Includes dismiss and session-based idle detection.

- **whats-new.json — structured release data**
  A dedicated JSON file for the current release with i18n support (en/th), typed sections (new/improved/fixed), and detailed change descriptions.

- **version.json — app-wide version tracking**
  A simple version file polled by the What's New page every 60 seconds to detect when a new update has been deployed.

### Improved

- **popup.js loaded on all 10 HTML pages**
  All site pages now include popup.js before version-core.js, ensuring the update notification system works correctly on every page.

- **What's New page with live relative timestamps**
  The What's New page now shows "Just now", "5 minutes ago", etc. with live ticking that automatically switches to full dates after 10 days. Supports both English and Thai.

### Fixed

- **Update notification popups not showing on any page**
  popup.js was not included in any HTML page, causing version-core.js to silently skip popup display after its 5-second wait timeout. Added the script tag to all 10 pages.
